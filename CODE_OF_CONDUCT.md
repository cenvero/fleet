# Code of Conduct

## Purpose

Cenvero Fleet should be a technically serious and welcoming project. We want a collaboration style that is direct, respectful, and useful to the people building, reviewing, testing, and operating the software.

## Scope

This code of conduct applies to project spaces including:

- the repository and pull requests
- issue discussions
- release and documentation discussions
- any project-managed community spaces tied to Cenvero Fleet

## Expected Behavior

Participants are expected to:

- be respectful, calm, and collaborative
- critique code, designs, and ideas without attacking people
- explain technical disagreements with evidence and context
- treat security, reliability, and operator risk as real concerns
- assume good intent until there is a reason not to
- help keep the project readable for newcomers and maintainers

## Unacceptable Behavior

The following behavior is not acceptable:

- harassment, intimidation, threats, or personal attacks
- discriminatory or degrading language
- deliberately derailing reviews or discussions
- sharing private security details irresponsibly
- doxxing, impersonation, or invasion of privacy
- knowingly submitting malicious code, misleading test evidence, or deceptive release material

## Maintainer Responsibilities

Maintainers are expected to:

- apply this policy consistently
- protect contributors and operators from abusive behavior
- distinguish between strong technical disagreement and personal misconduct
- act quickly when safety, privacy, or security are at risk

## Reporting

If you experience or witness unacceptable behavior:

- contact a project maintainer privately through the repository owner contact path on GitHub or another private maintainer channel
- do not open a public issue if doing so would create additional harm
- include links, screenshots, or context when possible

Security incidents should follow [SECURITY.md](SECURITY.md), not this file.

## Enforcement

Maintainers may take any action they believe is necessary to protect the project and its participants, including:

- asking for a behavior change
- moderating or removing comments
- closing discussions
- rejecting contributions
- temporarily or permanently restricting participation

## Interpretation

This code of conduct is intended to support a healthy working environment, not to suppress technical rigor. Strong technical feedback is welcome. Contempt, hostility, and reckless behavior are not.
