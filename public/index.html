Operator owned by design

Cenvero Fleet’s controller can live on a laptop, workstation, or server. Reverse mode handles NAT’d nodes when you have a stable listener. Direct mode works when the controller roams and the remote host exposes SSH. Mix both in one deployment.

Why Cenvero Fleet

Every core decision is built around a Unix-feeling binary, explicit ownership, and distribution that doesn’t collapse under its own packaging.

One static binary

Go-first packaging keeps the controller and agent easy to cross-compile, audit, and install.

Two transport modes

Run direct controller-to-agent sessions or persistent reverse connections from NAT’d nodes.

Operator-owned data

Config, keys, logs, backups, and state stay in the directory the operator chose during `fleet init`.

Modern crypto

Ed25519 by default, AEAD-only SSH ciphers, host key pinning, audit logs, and signed release plumbing.

Two modes, one controller

Reverse mode

Use this when the controller has a stable address and the remote node does not. The agent dials out and keeps the channel alive.

Direct mode

Use this when the controller is behind NAT or moves between networks. Cenvero Fleet reaches out to each server on demand.

Quick start

$ curl -fsSL https://fleet.cenvero.org/install | sh
$ fleet init
$ fleet server add web-01 192.0.2.10 --mode direct \
    --login-user root --login-key ~/.ssh/id_ed25519
$ fleet ssh web-01
$ fleet exec --all uptime
